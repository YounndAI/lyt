# Driving Lyt with AI agents

Lyt is built for agent-mediated operation. The harness skills (Claude Code,
Codex, MCP clients) call the CLI under the hood. Every new vault scaffolds an
`agents.md` priming file at its root telling the agent how to drive Lyt in
that vault's context.

## `agents.md` lifecycle

- Written once at `lyt vault init` from a templated v1 instruction block
  (`@AGENT_TEMPLATE version=1`).
- Bumped by `lyt vault sync-metadata --apply` if the template version drifts
  forward in a future Lyt release.
- The "Installed patterns + skills" section is empty in v1.0.0; populated by
  Phase 7F when `lyt pattern *` ships.

## Etiquette agents should follow

- Read `agents.md` before editing.
- Read `.lyt/vault.yon` to learn the vault's mesh edges.
- Tag new Figments with frontmatter `tags:` arrays — federated indexes use them.
- Use `[[wikilinks]]` for cross-Figment and cross-vault refs.
- Never edit `.lyt/indexes/*.db` (libSQL derived state at `.lyt/indexes/lyt.db` + `.lyt/indexes/audit.db` + `.lyt/indexes/provenance.db` post-v1.A.2c DB SPLIT; regenerated by `lyt vault rebuild-index`).
- Never `git add -A`; always name paths.
- After substantive edits, commit + push (Phase 8 will provide `lyt sync`).

## v1.0.0 daemon UX cliff (per `lyt help troubleshooting`)

Phase 11 ships `lyt agent run` as a **foreground** daemon. Closing the
terminal stops the agent. Workarounds: `nohup`, `launchd`, Windows Task
Scheduler. The next CLI command an agent runs (`lyt sync`, `lyt vault verify`,
`lyt mesh status`, `lyt doctor`) reports the last-stop time if the agent isn't
running, so a stop is discovered on the next interaction.

The full background daemon ships in v1.1 via `lyt agent start/stop/status/logs`.

## Skill files

`@younndai/lyt-skills` ships skill files for the installed harness. v1.0.0
ships `/lyt-capture` + `/lyt-recall`. Phase 7F adds eight more wrapping the
work-management + knowledge-capture + decision-log patterns.

## Forward-doc verbs (Phase 7F)

`lyt pattern list/install/uninstall/link/unlink/fork/verbs/run` ship in Phase
7F. The bundled patterns (`work-management`, `knowledge-capture`,
`decision-log`, `project-lifecycle`) install to `~/lyt/patterns/` via the
package's postinstall step.
